# Game-style Patterns — Index

> 12 个核心 gamification pattern。每个 pattern = **intent / structure / when-to-use / 它替代掉的 slop 或 dark-pattern**。
>
> 这些是**机制结构**，不是视觉皮肤 —— 颜色 / 字体 / 圆角由锁定的 L3 风格决定（见 README "它和 L3 怎么 COMPOSE"）。motion 手感见 `motion-tokens.md`，state 覆盖见 `interaction.md`。

## 1. Progress Ring

- **intent**：把"今天这件事完成了多少"压缩成一个一眼可读的闭环。Apple Fitness 三环是这个 pattern 的黄金范例 —— 闭环 = 达成，是强目标梯度。
- **structure**：SVG circle（`stroke-dasharray` + `stroke-dashoffset` 驱动），中心放当前值 / 目标值。多环嵌套时半径递减、各自独立闭合。闭合瞬间有一个"咔哒"的 pop（ring 微胀后回弹）。
- **when-to-use**：单一或少数（≤3）可量化日目标（步数 / 卡路里 / 学习分钟）。目标固定、每日重置。
- **替代掉**：一个孤零零的 `73%` 文本 —— 它没有目标梯度、没有"就差一点"的拉力、闭合时没有奖励感。Ring 把同样的数据变成"我想把它填满"的冲动。

## 2. Progress Bar (segmented)

- **intent**：表达"分 N 步 / 分 N 节，现在在第几"。比连续条更适合**离散里程碑**（课程 5 单元、onboarding 4 步）。
- **structure**：N 个分段 cell，已完成段实心、当前段半亮 + 微脉冲、未来段留空。整体可叠一个细的总进度。段切换是**阶跃**不是平滑滑动（强调"跨过了一格"）。
- **when-to-use**：步骤有明确边界、用户需要知道"还剩几步"。课程、表单向导、setup quest。
- **替代掉**：灌水的伪进度条（一进页面就 30% / 永远在 99% 卡住逼充值）。Segmented bar 每一段都对应**真实完成的事**，不许有"凭空起步"的水分（铁律 2）。

## 3. Streak Counter

- **intent**：用"连续 N 天"放大坚持的心理重量 —— 损失厌恶让人不想打破连续。Duolingo 的火苗是教科书级别。
- **structure**：大号数字 + 火苗 / 日历点阵图标。+1 时数字**翻牌**（旧数字上滚出、新数字下滚入），火苗做一次脉冲。可附 7 日点阵显示本周打卡。
- **when-to-use**：日频 / 高频习惯，希望用连续性驱动留存。
- **替代掉**：**断了就无情清零 + 红字恐吓**的 dark pattern。本 pattern 强制配 **streak freeze / 补签**恢复机制（铁律 7）—— 激励坚持，但断了给温柔的台阶下，不羞辱、不用焦虑逼复购。

## 4. XP / Level Meter

- **intent**：把零散动作累积成"我在变强"的长期成长叙事。等级是身份感，XP 是即时的"我刚做的事有意义"。
- **structure**：当前等级徽章 + 一条朝下一级的 XP bar（显示 `当前/升级所需`）。获得 XP 时 bar 增量填充；填满 → 溢出动画 → level-up 庆祝（见 pattern 11）→ bar 归零进下一级。
- **when-to-use**：长期参与产品，需要超越"单日目标"的累积动机。学习、健身、社区贡献。
- **替代掉**：**满天飞的无意义数字** —— 到处 +5 +10 但用户不知道换来什么。XP 必须**真的能换到东西**（解锁内容 / 等级 / 特权），否则就是噪音 slop。

## 5. Achievement / Badge

- **intent**：把"做到了一件值得纪念的事"固化成可收藏的徽章。稀缺性 = 价值；满墙的徽章 = 贬值。
- **structure**：徽章网格，已解锁彩色 + 未解锁灰阶剪影（剪影暗示"存在但没拿到"，制造目标）。解锁瞬间：剪影 → 彩色的翻转 + 一次性微光扫过（≤1 次，铁律 5）。
- **when-to-use**：有清晰的里程碑 / 成就体系（首次、第 100 次、全勤、隐藏挑战）。
- **替代掉**：**一进门就发"欢迎奖 / 参与奖"**的廉价通胀。Badge 必须对应**真实达成**，且未解锁态可见（灰剪影）才有目标拉力 —— 全亮或全藏都失去意义。

## 6. Reward Reveal

- **intent**：用"揭晓前的悬念"放大奖励的愉悦（anticipation 是 juice 的核心）。拆盒 / 翻牌 / 开箱那一下的张力。
- **structure**：三段式 —— ① anticipation（盒子轻微抖动 / 蓄力下压 squash）② reveal（开启，内容弹出 overshoot）③ settle（回稳 + 内容入位）。全程 ≤ 900ms。
- **when-to-use**：每日奖励、任务完成奖、里程碑解锁内容。需要"惊喜感"的奖励交付。
- **替代掉**：**老虎机 / loot box 式的随机付费抽卡赌博**。本 pattern 的奖励必须是**已挣得、确定性**的揭晓（用户知道自己赢了什么类别），用悬念增加愉悦感 —— 而**不是**用随机性 + 付费制造赌博成瘾（这是 dark pattern 红线，铁律 3）。

## 7. Daily Goal

- **intent**：给"今天该做什么"一个明确、可达成、当日重置的聚焦点。完成日目标是每日的小高潮。
- **structure**：醒目的当日目标卡（目标 + 当前进度，常配 progress ring/bar）。达成瞬间触发**克制的**庆祝（pattern 11 的轻量版）。次日重置。
- **when-to-use**：日频产品的首屏核心。健身今日环、学习今日 N 分钟、冥想今日 1 次。
- **替代掉**：**永远完不成 / 模糊到无法达成的目标** —— 让用户长期处于"没做到"的挫败。Daily goal 必须**可在合理努力内达成**，达成率设计成大多数日子能闭环，否则激励变惩罚。

## 8. Leaderboard

- **intent**：用社交对比驱动竞争动机。Duolingo 的联赛（按段位分组、周期晋降级）是把"对比"做得健康的范例。
- **structure**：排名列表（头像 + 名字 + 分数 + 名次变化箭头）。当前用户行高亮固定可见。排名变化用滑动重排动画。**分组 / 分段位**（而非全球绝对榜）让每个人都有可竞争的同温层。
- **when-to-use**：有社交 / 竞争属性、用户愿意被看到的产品。注意：内向型 / 健康类要给**关闭选项**。
- **替代掉**：**全球绝对值羞辱式排名**（永远垫底、毫无希望）。健康做法是**段位分组 + 周期重置 + 晋降级**，让对比发生在势均力敌的小组内，给希望而非碾压。

## 9. Combo / Multiplier

- **intent**：对"连续做对 / 连续命中"给即时的、放大的正反馈。连击让"做对"这件事本身变爽。Monkeytype 的连续正确、答题连对都是。
- **structure**：连击计数 + 倍率标签（×2 ×3）。每次累加：数字**弹跳缩放**（scale overshoot）+ 颜色随热度升温。断连：快速回落归零（不羞辱，只是回到基线）。
- **when-to-use**：高频、快节奏、有"对错"判定的交互（答题、打字、节奏）。
- **替代掉**：**用连击焦虑驱动强迫性行为**（怕断连而无法停止）。Combo 应该是"做对的额外糖"，断了**轻描淡写**地归零，不做戏剧化的失败惩罚动画，不让用户为了保连击而被困住。

## 10. Lives / Energy

- **intent**：用资源限制制造节奏感和"每次机会都珍贵"的张力（答错扣命、行动消耗体力）。
- **structure**：心 / 闪电图标行。消耗：一个图标"破碎 / 熄灭" + 轻微心跳。回充：随时间或行为逐个点亮（带回充倒计时）。
- **when-to-use**：希望控制节奏 / 防止盲目刷题、给失败一点点（但不致命）后果的学习类产品。
- **替代掉**：**体力耗尽 → 弹窗逼内购续命**的 pay-to-continue dark pattern。健康做法：体力**靠时间自然回满**是默认路径，付费加速是可选**便利**而非**强制门槛**；耗尽不羞辱、不打断到无法使用核心功能（铁律 3）。

## 11. Milestone Celebration

- **intent**：在**真正重要**的节点（升级 / 完成大目标 / 解锁成就）给一个情绪高潮 —— 这是 juice 的回报顶点。confetti / level-up 属于这里。
- **structure**：分层 —— 核心信息（"Level 5!" / "目标达成"）+ 一次性 confetti burst（canvas-confetti，单次、有粒子上限）+ 主体元素的 overshoot 入场。全程 ≤ 1200ms，**一个动作只触发一次**。
- **when-to-use**：**稀疏**地用在高价值时刻。日常小成就用轻量反馈即可，把全套庆祝留给真正的里程碑 —— 滥用 = 廉价。
- **替代掉**：**全屏霸占 + 连发 + 廉价热闹**的 slop（每点一下都 confetti、彩虹满天、卡通音效轰炸）。铁律 1+5 强制：≤1200ms、≤1次/动作、confetti ≤ 一次 burst（≤80-150 粒）、不阻塞下一步输入。**克制**才让庆祝有分量。

## 12. Quest / Checklist

- **intent**：把多步引导 / 多任务拆成一串可逐个攻克的小任务，每打一勾都是微进步。onboarding quest、每周挑战清单。
- **structure**：任务列表，每项 = 图标 + 标题 + 完成态（勾 / 进度）。完成一项：打勾动画（draw-in 勾 + 该行轻微下沉确认）+ 顶部总进度收束。全部完成触发 milestone celebration。
- **when-to-use**：onboarding、多步设置、周期性挑战包。需要把"大而模糊的开始"拆成"小而明确的下一步"。
- **替代掉**：**一次性塞满 20 项的待办墙** —— 让新用户一进来就被淹没（违反目标梯度）。Quest 应**渐进暴露**（先给 3-5 项、完成后解锁下一批），每一项都是"明确的下一步"而非"无穷的清单焦虑"。
