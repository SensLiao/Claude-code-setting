# Game-style — 八条铁律（gate 形式）

> 任何 game-style 相关 artifact 必须每条都打勾才放行。这不是"建议清单"，是 **Stage 3 红队 gate**。
>
> 这八条里有两类：**手感/性能铁律**（1/4/5/6 —— 保证 juice 是高质量的）和**伦理铁律**（2/3/7 —— 保证 gamification 不变成操纵）。**伦理铁律一票否决，没有"风格例外"。**

## 速查 checklist

- [ ] **铁律 1**：庆祝有上限（≤1200ms 完整、≤1 次/动作、不连发、不全屏霸占）
- [ ] **铁律 2**：诚实进度（进度反映真实状态、零灌水、不在 99% 卡住逼充值）
- [ ] **铁律 3**：没有 fake urgency / dark pattern（无 fake countdown、无 streak 勒索、无 pay-to-continue 逼内购、无 loot-box 赌博）
- [ ] **铁律 4**：juice 永不阻塞输入（庆祝/弹跳期间下一动作立即可点、动画可打断）
- [ ] **铁律 5**：粒子有预算（confetti ≤ 一次 burst ≤ 80-150 粒、≤ 2s；常驻 shimmer/glow 每屏 ≤ 1 处）
- [ ] **铁律 6**：motion-sensitive 兜底（`prefers-reduced-motion` 保留反馈语义、去弹跳/粒子/抖动）
- [ ] **铁律 7**：streak 不绑架（断了给 streak freeze / 补签、不羞辱、不用损失厌恶逼复购）
- [ ] **铁律 8**：反 childish slop（不堆 emoji、不彩虹满屏、不音效轰炸；juice 是克制的手感）

---

## 铁律 1 · 庆祝有上限

**gate 数字**：

| 项目 | 上限 |
|------|------|
| 单次庆祝动画总时长 | ≤ **1200ms**（confetti 余韵另算，但主信息 ≤1200ms） |
| 同一动作触发庆祝次数 | ≤ **1 次** |
| 庆祝占屏面积 | 不全屏霸占；overlay 必须可一键 / 自动消散，核心内容不被长时间遮挡 |
| 日常小成就 | 用轻量反馈（≤400ms），**不**触发全套庆祝 |

**判失败**：升级动画跑了 2.5s / 每答对一题都放 confetti / 庆祝 overlay 不消失挡住下一题。

## 铁律 2 · 诚实进度

进度条 / ring / XP / streak 的数值**必须映射真实状态**：

- ❌ 一进页面就显示 30%（无中生有的"起步水分"）
- ❌ 进度永远卡在 99%，差最后 1% 要付费 / 看广告
- ❌ "差一点点"是假的（实际还差一大截，UI 谎称就快了）
- ✅ 每一格 / 每一度填充都对应**真实完成的动作**
- ✅ "就差 X" 是真的就差 X

**判失败**：任何让用户**误判自己离目标的距离**的进度表达。这是信任红线。

## 铁律 3 · 没有 fake urgency / dark pattern

**禁止清单**（命中任意一条 = 直接驳回）：

- **fake countdown** —— 倒计时结束又重置 / 永远"仅剩 3 分钟"的虚假紧迫
- **streak 焦虑勒索** —— "再不打卡就全没了！"式恐吓文案 + 红色警告轰炸（streak 机制本身 OK，勒索式表达不 OK，配套见铁律 7）
- **pay-to-continue** —— lives/energy 耗尽 → 弹窗逼内购续命才能用核心功能
- **loot-box 赌博** —— 付费 + 随机结果的抽卡机制（reward reveal 必须是确定性的已挣得奖励，见 pattern 6）
- **虚假稀缺** —— "限量名额""仅此一次"但实际无限供应
- **confirmshaming** —— 拒绝按钮写成"不，我不想变得更好"式羞辱

**判失败**：任何用焦虑 / 恐惧 / 赌博成瘾 / 羞辱来驱动行为的机制。**内在动机（成长、掌控、好奇）是合法燃料；操纵情绪是 dark pattern。**

## 铁律 4 · juice 永不阻塞输入

- 用户点"完成" → 庆祝开始播 → **下一个动作此刻就必须可点**，不能等动画跑完
- confetti / 弹跳 / 角色动画期间，pointer event / 键盘输入**绝不冻结**
- 任何 juice 动画必须可被打断（用户操作 → 动画立即让路）
- 答题 / 打字这类高频场景，反馈动画**绝不能让输入丢帧 / 丢字**

**判失败**：庆祝动画期间按钮变灰 / 点击无响应 / 打字卡顿丢字。**手感 ≠ 拖慢**。

## 铁律 5 · 粒子有预算

| 项目 | 上限 |
|------|------|
| confetti 单次触发 | **一次 burst**，建议 **≤ 80-150 粒** |
| confetti 持续 | ≤ **2s**（含飘落余韵） |
| 常驻装饰（shimmer / glow / 浮动粒子背景） | 每屏 **≤ 1 处** |
| 同屏同时运行的庆祝级动画 | ≤ **1 个** |

**判失败**：confetti 几千粒拖垮帧率 / 多个庆祝叠加 / 满屏常驻闪光让人眼花。**克制的粒子才有"高级感"，泛滥的粒子是廉价。**

## 铁律 6 · motion-sensitive 兜底

`prefers-reduced-motion: reduce` 下，每个 juice 动效必须有兜底，且**保留反馈语义**：

- ✅ **保留**：数字变了、勾出现了、色块从灰变亮、进度填充了（用 ≤150ms opacity/color 过渡完成，不跳变）
- ❌ **去掉**：弹跳 overshoot、squash/stretch、confetti / 粒子、屏幕抖动 / shake、角色蹦跳

**核心原则**：reduced-motion 用户**仍然要知道"我成功了 / 我进步了"** —— 只是用安静的方式告诉他。不许"reduced 模式下什么反馈都没有"。

**判失败**：reduced-motion 下庆祝完全消失（用户不知道发生了什么）/ 仍然播 confetti / 仍然抖屏。

## 铁律 7 · streak 不绑架

streak 是强力激励，但**不许用它绑架用户**：

- ✅ 必须配**恢复机制**：streak freeze（可消耗的"护盾"）/ 补签 / 宽限日
- ✅ 断了的文案**温柔**："休息一天也没关系，明天继续" 而非 "你毁掉了 47 天连续！"
- ❌ 不用损失厌恶 + 红字恐吓逼用户每天回来 / 逼复购续费保 streak
- ❌ 不在用户明显需要休息时（深夜 / 连续高强度）还推焦虑通知

**判失败**：streak 断了用羞辱 / 恐吓 / 损失轰炸；没有任何恢复台阶。**激励坚持 ≠ 制造焦虑依赖。**

## 铁律 8 · 反 childish slop

gamification ≠ 幼稚。守住"成熟的玩感"：

- ❌ emoji 当主要视觉语言堆砌（🎉🔥⭐ 满屏）
- ❌ 彩虹渐变 / 高饱和原色满屏（除非 L3 风格明确如此，且那是 L3 的决定不是 archetype 的）
- ❌ 卡通弹簧音效 / 欢呼声轰炸
- ❌ 把所有元素都做成果冻弹跳（squash/stretch 是调味，不是全局默认）
- ✅ juice 是**克制的、有物理可信度的手感**（参考 Apple Fitness / Headspace / Cash App —— 它们有玩感但不幼稚）

**判失败**：界面看起来像给 5 岁小孩做的 / 廉价手游氪金弹窗。**目标范例是 Duolingo 的"专业的可爱"，不是山寨小游戏的"廉价的热闹"。**

---

## 与通用 anti-patterns 的关系

本 gate 是 game-style **专项**，叠加在 `references/anti-patterns.md` 通用红队之上，**不替代**它。通用动效铁律（>500ms 普通动画、动 layout-bound 属性、无 reduced-motion）仍然生效 —— 注意 game-style 给**庆祝类**动画放宽到 ≤1200ms（铁律 1），但**非庆祝**的常规状态切换仍守通用的 ≤500ms。
