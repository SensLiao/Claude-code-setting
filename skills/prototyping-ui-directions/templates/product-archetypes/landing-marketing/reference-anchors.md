# Reference Anchors — Landing-Marketing

> 8 个 real gold-standard 营销 / 着陆页范例。Stage 1 reference-acquisition 消费本文件：按用户产品调性选 2-4 个去**实地研究真实页面**(截图 / 抓 section 结构 / 记 CTA 节奏)，把"真实设计智慧"喂进 Stage 2，而不是用 AI 默认套路。
>
> 每条 = **name** + **where to find** + **why exemplary** + **the ONE thing to study**(只学这一件最有信号的事，不要泛泛照抄)。
>
> ⚠️ 这些是**结构 / 转化智慧**的来源，不是视觉皮肤来源。研究它们的 section 顺序、CTA 节奏、proof 结构、信息层级——但最终视觉由锁定的 L3 风格(taste / luxury / brutalist)决定，**不照搬它们的具体配色/字体**。
>
> 🕐 真实页面会随 redesign 变化。研究时以**当前线上版本**为准；本文件描述的是各家**稳定的设计哲学**，不是某次具体版式的逐像素快照。

## 1. Linear — `linear.app`

- **where to find**：主页 `linear.app`；feature 子页 `linear.app/features`；变更日志 `linear.app/changelog`(launch 页范本)。
- **why exemplary**：被公认为 SaaS 营销页的"黄金标准"——克制的深色质感、精准的信息层级、产品截图主导、近乎完美的 above-fold 纪律。每一屏都在卖产品本身，零废话。
- **the ONE thing to study**：**hero 的 above-fold 纪律 + 单一 value prop**——一句强 headline + 一句 subhead + 单个 primary CTA + 真实产品截图，没有任何堆叠。看它如何用 typography scale 和留白(而非渐变球)制造气场。对应 `patterns-index.md` #3 Product-in-Hand hero + 铁律 1/3。

## 2. Stripe — `stripe.com`

- **where to find**：主页 `stripe.com`；产品页 `stripe.com/payments`；定价 `stripe.com/pricing`。
- **why exemplary**：把"抽象基础设施(支付 API)"讲成可感知价值的教科书——分层叙事、代码片段直接当视觉证据、客户 logo 与 metric 的信任堆叠、渐变与动效极度克制且服务内容。
- **the ONE thing to study**：**social-proof 的分层堆叠 + 抽象产品的可视化**——看它怎么把 API 这种"看不见的东西"用真实代码块 + 客户 logo bar + 具体 metric 组合成信任链。对应 `patterns-index.md` #2 Centered Statement / #4 social-proof bar / #6 metric strip。

## 3. Vercel — `vercel.com`

- **where to find**：主页 `vercel.com`；`vercel.com/home`；产品页 `vercel.com/products/*`。
- **why exemplary**：深色技术美学的标杆——高对比黑白 + 单一强调色、bento feature grid 的现代编排、性能本身(快)就是品牌。开发者工具营销的范式。
- **the ONE thing to study**：**bento feature grid 的主次编码**——看它如何用大小不一的 cell 在一屏内同时传达"平台很全 + 哪个 feature 最重要"，用尺寸差异制造层级而非等高卡阵列。对应 `patterns-index.md` #8 bento + `layout-engines.md` bento 段。

## 4. Raycast — `raycast.com`

- **where to find**：主页 `raycast.com`；`raycast.com/pro`(定价/升级页)。
- **why exemplary**：产品截图与短动效驱动的 hero 典范——把一个"命令面板工具"的真实交互直接搬上首屏，showing-beats-telling 执行到极致；深色质感精致但不喧宾夺主。
- **the ONE thing to study**：**用真实产品交互动效当 hero 视觉锚**——看它如何用一段真实操作的 loop(而非抽象插画)在 3 秒内证明"这个工具用起来什么感觉"。注意动效**不挡** CTA、可立即点击。对应 `patterns-index.md` #3 Product-in-Hand + `motion-tokens.md`(hero entrance 不阻塞)。

## 5. Framer — `framer.com`

- **where to find**：主页 `framer.com`；模板/feature 页 `framer.com/features/*`。
- **why exemplary**：用自家产品做自家营销页(dogfooding)——scroll-reveal 与 entrance 动效丰富但克制，section pacing 节奏感强，是"motion 增强转化而非干扰"的活样本。
- **the ONE thing to study**：**scroll-reveal 序列与 section pacing 的节奏**——看它如何用 fade-up / stagger 让每个 section 进场有呼吸感，同时**绝不**把页面做成逐帧 scrollytelling(到达 CTA 始终很快)。对照本 archetype 与 `narrative-scrolly` 的边界。对应 `motion-tokens.md` scroll-reveal 段 + 铁律 9。

## 6. Clerk — `clerk.com`

- **where to find**：主页 `clerk.com`；定价 `clerk.com/pricing`；feature 页 `clerk.com/features/*`。
- **why exemplary**：开发者认证产品的高完成度营销页——feature-row 交替讲透每个能力、定价页锚定清晰、proof 与 CTA 节奏教科书级。
- **the ONE thing to study**：**feature-row zig-zag 的"逐个讲透"**——看它如何用图文左右交替给每个核心 feature 充分篇幅 + 真实 UI 截图，而不是塞一个 3-up 浅 icon grid。注意每行的 micro-CTA / 链接如何延续转化动线。对应 `patterns-index.md` #7 feature-row alternation。

## 7. Resend — `resend.com`

- **where to find**：主页 `resend.com`；定价 `resend.com/pricing`；文档入口 `resend.com/docs`。
- **why exemplary**：极简开发者工具营销页的范本——克制、快、信息密度恰到好处；定价页清晰锚定(免费档 + 明确付费档),把"邮件 API"讲得简单可信。
- **the ONE thing to study**：**pricing table 的锚定与清晰度**——看它如何用"明确免费档 + 明确价格付费档 + 单一推荐"降低决策摩擦，而不是三列全 "Contact us"。注意计费周期呈现与 feature 清单的精简。对应 `patterns-index.md` #10 pricing + `interaction.md` pricing 段。

## 8. Superhuman — `superhuman.com`

- **where to find**：主页 `superhuman.com`；feature/定价 `superhuman.com/pricing`。
- **why exemplary**：高端定位(premium email)的转化页——用速度感 + 精英背书 + 强情感文案建立"这值这个价"的叙事；testimonial 与 metric 的信任组合精准。
- **the ONE thing to study**：**testimonial / 情感背书与高端定位的对齐**——看它如何用**真实署名**的客户 quote + 具体结果("save hours")承接转化，把一个高价产品的"凭什么"讲清。注意它如何避免匿名假评价。对应 `patterns-index.md` #11 testimonial + 铁律 4(proof 真实性)。

## 选择指引（Stage 1 用）

按用户产品调性 + 重点 dimension 选 2-4 个去研究，**不要全抄**：

| 用户产品类型 / 重点 | 优先研究 |
|---------------------|---------|
| 有精致 UI 的工具(showing 强) | Linear, Raycast, Superhuman |
| 抽象平台 / 基础设施 / API | Stripe, Vercel, Resend |
| feature 多、要讲透 | Clerk(zig-zag), Vercel(bento) |
| 定价是关键转化关口 | Resend, Clerk, Stripe |
| 想要丰富但克制的 motion | Framer, Linear, Raycast |
| 高端 / premium 定位 | Superhuman, Linear |
| social-proof / 信任是命门 | Stripe, Superhuman, Clerk |

## 研究方法（喂给 Stage 2 extract card）

对每个选中范例，提取并记录(对应 `patterns-index.md` 的结构)：

1. **section 序列**——从上到下依次是哪些 section？顺序如何？(对照本 archetype 默认序列模板)
2. **hero 变体**——split / centered / product-in-hand 哪种？value prop 几个词？视觉锚是什么？
3. **CTA 节奏**——全页 primary CTA 出现几次？间隔多少屏？文案是否一致？
4. **proof 类型与位置**——logo bar / metric / testimonial / comparison 哪些？放哪？真实性如何保证？
5. **F/Z 动线**——关键信息落在视线哪个节点？
6. **motion 用法**——reveal 怎么做？有无 parallax？到达 CTA 快不快？

> **铁律**：研究是为了**借结构智慧**，不是 clone。提取的是"营销页该怎么搭骨架 + 怎么排转化节奏"，最终套用户锁定的 L3 视觉风格。把范例的具体配色/字体照搬 = 误用本文件。
